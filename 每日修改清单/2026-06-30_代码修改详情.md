## 三、修改文件清单及详细变更

### 3.1 后端 - 简历优化模块

#### 3.1.1 ResumeTextExtractor.java（新增）

**路径**：`软件实训项目-code/backend/src/main/java/com/xuelian/career/util/ResumeTextExtractor.java`

**核心方法**：

| 方法名 | 说明 | 参数 | 返回值 |
|---|---|---|---|
| `extractText(MultipartFile file)` | 从上传文件提取文本 | `MultipartFile file` - 上传的简历文件 | `String` - 提取的纯文本 |
| `extractTextFromPath(Path path, String ext)` | 从本地路径提取文本 | `Path path` - 文件路径；`String ext` - 扩展名 | `String` - 提取的纯文本 |
| `extractPdf(Path path)` | PDF 文本提取（内部） | `Path path` | `String` |
| `extractDocx(Path path)` | DOCX 文本提取（内部） | `Path path` | `String` |
| `extractTxt(Path path)` | TXT 文本提取（内部） | `Path path` | `String` |
| `cleanText(String text)` | 文本清洗（去重空行等） | `String text` | `String` |

**依赖**：Apache PDFBox、Apache POI（已有依赖）

---

#### 3.1.2 FileUtil.java（修改）

**路径**：`软件实训项目-code/backend/src/main/java/com/xuelian/career/util/FileUtil.java`

**变更点**：

1. **新增 TXT 格式支持**
   - `ALLOWED_TYPES` 新增 `"text/plain"`
   - `ALLOWED_EXTENSIONS` 新增 `"txt"`

2. **新增方法**：
   - `resolveFilePath(String relativePath)` - 根据相对路径解析绝对路径
   - `extractResumeText(String relativePath)` - 从已上传文件中提取简历文本

---

#### 3.1.3 ResumeServiceImpl.java（重大修改）

**路径**：`软件实训项目-code/backend/src/main/java/com/xuelian/career/service/impl/ResumeServiceImpl.java`

**核心变更**：

1. **analyzeResume 方法重构**
   - 提取真实简历文本（之前传的是文件路径）
   - 查询目标岗位及技能要求
   - 调整 AI 调用参数：超时 30s、max_tokens 4096、temperature 0.3
   - 后处理增强：JSON 解析、字段校验、兜底填充

2. **新增方法**：
   - `buildOptimizedResume(String resumeText, List<IssueItem> issues, JobPosition job)`
     - 生成默认优化简历全文
     - 当 AI 未返回 optimizedResume 或走规则兜底时调用
     - 包含：个人信息、求职意向、教育背景、技能清单、项目经历 STAR 模板、自我评价、优化方向

3. **fallbackAnalysis 增强**
   - 基于简历文本长度、技能关键词匹配动态评分
   - 生成规则兜底优化简历文本

### 3.2 后端 - 职业探索模块

#### 3.2.1 CareerExplorationServiceImpl.java（重大修改）

**路径**：`软件实训项目-code/backend/src/main/java/com/xuelian/career/service/impl/CareerExplorationServiceImpl.java`

**核心变更**：

1. **信息充足度校验**
   - 新增 `isInformationInsufficient()` 方法
   - 判断条件：无兴趣岗位 + 无城市 + 无测评结果
   - 信息不足时调用 `askForClarification()` 生成追问响应

2. **追问模式**
   - 新增 `askForClarification()` 方法
   - 返回 `needClarification=true` + 自然语言追问文案
   - 引导用户补充兴趣、城市或完成测评

3. **测评场景兴趣回退修复**
   - `extractInterestCity()` 新增 `preferAssessment` 参数
   - 测评场景（source=ASSESSMENT）禁止兴趣回退到 profile.targetRoles
   - 改用测评优势维度映射为兴趣（如编程能力最高→后端）

4. **双队列语义修复**
   - `isInterestMatched()` 空值判断从 true 改为 false
   - 兴趣为空时所有岗位进入 fallback 队列

5. **后处理增强**
   - `postProcessRecommendations()` 强制设置 `needClarification=false`
   - 防止 AI 误返回追问标记

## 四、接口变更说明

### 4.1 简历分析接口

**接口**：`POST /api/student/resume/analyze`

**请求**：无变化

**响应新增字段**：

```json
{
  "code": 200,
  "data": {
    "score": 75,
    "dimensionScores": {...},
    "issues": [...],
    "optimizedSnippets": [...],
    "summary": "...",
    "optimizedResume": "【AI 优化版简历】\n\n【个人信息 / 求职意向】...",
    "source": "AI",
    "createdAt": "2026-06-30T17:00:00"
  }
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `optimizedResume` | String | AI 生成的完整优化简历全文，可能为 null（走 AI 兜底时由服务端生成） |

| 参数 | 类型 | 说明 |

**响应新增字段**：

| 字段 | 类型 | 说明 |
|---|---|---|
| `needClarification` | Boolean | 是否需要用户补充信息（true 时展示追问文案） |

## 六、验证方式

### 6.1 简历优化功能验证

```bash
# 1. 上传简历
curl -X POST http://localhost:8080/api/student/resume/upload \
  -H "Authorization: Bearer <token>" \
  -F "file=@test_resume.pdf"

# 2. 分析简历
curl -X POST http://localhost:8080/api/student/resume/analyze \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"fileUrl":"resumes/xxx/test_resume.pdf"}'

# 验证点：
# - 返回 data.optimizedResume 不为空
# - source=AI 时为 AI 生成内容
# - source=RULE_FALLBACK 时为规则兜底内容
```
